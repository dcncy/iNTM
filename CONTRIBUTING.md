# 反馈与建议

感谢你使用 iNTM！我们欢迎各种形式的反馈，包括：

- 🐛 报告 Bug
- 💡 提出新功能建议
- 📝 改进文档建议

---

## 🐛 报告 Bug

### 提交 Bug 报告前

1. **检查现有 Issues**：确保该问题尚未被报告
2. **使用最新版本**：确认你使用的是最新版本的 iNTM
3. **收集信息**：准备好复现步骤、系统信息和日志

### 如何提交

1. 前往 [Issues 页面](https://github.com/yourusername/iNTM/issues/new)
2. 选择 "Bug Report" 模板
3. 填写以下信息：
   - **问题描述**：简明扼要地描述问题
   - **复现步骤**：详细的步骤，让我们能够重现问题
   - **期望行为**：描述你期望发生什么
   - **实际行为**：描述实际发生了什么
   - **环境信息**：
     - macOS 版本
     - iNTM 版本
     - 处理器类型（Apple Silicon / Intel）
   - **截图**：如果适用，添加截图帮助说明问题
   - **日志**：附加相关日志（可在控制台应用中查看）

---

## 💡 功能建议

### 提交功能建议前

1. **检查现有 Issues**：确保该功能尚未被提议
2. **思考必要性**：确认该功能对大多数用户有价值
3. **考虑实现**：思考该功能如何与现有功能协同

### 如何提交

1. 前往 [Issues 页面](https://github.com/yourusername/iNTM/issues/new)
2. 选择 "Feature Request" 模板
3. 填写以下信息：
   - **功能描述**：清晰描述你想要的功能
   - **使用场景**：解释为什么需要这个功能
   - **建议方案**：如果有，描述你认为应该如何实现
   - **替代方案**：是否有其他替代方案
   - **附加信息**：任何其他相关信息

## 📝 文档建议

### 改进文档

如果你发现文档中有错误或不清楚的地方，欢迎提出建议：

- 修正拼写或语法错误
- 改进措辞，使其更清晰
- 补充遗漏的信息
- 建议新增的文档主题

可以通过 Issues 提交文档改进建议。
